# Pass 1: Critical Checklist (BLOCKERS)

This pass focuses entirely on system-breaking bugs, security flaws, and architectural integrity. Any finding in this category MUST be classified as a **BLOCKER** and results in a `REQUEST CHANGES` verdict.

## 1. SQL & Data Safety

Direct database interactions require extreme scrutiny. Flag the following immediately:

- **String Interpolation in SQL:** Flag any raw SQL interpolation (even if values are `.to_i`/`.to_f`). Demand parameterized queries (e.g., Rails: `sanitize_sql_array`, Node: prepared statements, Python: parameterized queries).
- **Bypassing Model Validations:** Flag direct DB writes that bypass application-level validations (e.g., Rails: `update_column`, Django: `QuerySet.update()`, Prisma: raw queries).
- **N+1 Query Introduction:** Look for lazy loading inside loops. Demand eager loading or batching.
- **Direct Schema Mutations:** Flag schema changes in application code instead of proper migration files.

## 2. Race Conditions & Concurrency

Look for flaws in how state is mutated under concurrent load:

- **TOCTOU (Time-of-Check to Time-of-Use):** Flag check-then-set patterns that should be atomic (e.g., `WHERE` + `update_all` instead of `if (x) update(x)`).
- **Missing Locks:** Flag lack of pessimistic/optimistic locking when mutating shared financial or critical state.
- **Background Job Idempotency:** If a job fails halfway, can it safely retry? Flag jobs that mutate state without transactions or idempotency keys.

## 3. LLM Output & Trust Boundaries

Treat all outputs from Large Language Models or external APIs as malicious user input.

- **Unvalidated Output:** Flag any LLM output that is parsed, executed, or persisted without strict schema validation (e.g., Zod, Pydantic).
- **Prompt Injection:** Flag prompts that concatenate raw user input without XML tags or strict boundary delimiters.
- **Raw `eval()` or Execution:** Flag any system that directly executes code generated by an LLM without a sandboxed environment.

## 4. Enum & Value Completeness

When the diff introduces a new state, enum value, status, or tier:

- You MUST use your tools (`grep_search` or `read_file`) to check files _outside_ the diff.
- Verify that existing `switch` statements, frontend mappings, or database constraints handle the newly introduced value. Failure to handle new enums globally is a critical blocker.
